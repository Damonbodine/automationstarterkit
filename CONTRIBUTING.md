# Contributing Guide

Thanks for your interest in contributing!

## Ways to Help

- Triage issues and propose fixes
- Improve documentation
- Submit features and bug fixes

## Development Setup

- Node.js 22+
- `npm ci`
- `npm run dev` to start the app; `npm run workers` for background workers

## Testing & Quality

- Run `npm run lint` and ensure `tsc --noEmit` passes
- Add tests when feasible; keep changes focused

## Pull Requests

- Fork and branch from `main`
- One logical change per PR
- Describe the problem, solution, and any tradeoffs
